# E-Commerce-Funnel-Analysis-SQL-Power-BI
This project analyzes user behavior across an e-commerce funnel to identify conversion bottlenecks, revenue drivers, and high-value customer segments.

🎯 Business Problem

Despite strong traffic, the business faces:

Low conversion rate (~10%)
High drop-offs at the checkout stage
Unclear revenue contribution across segments

🧠 Approach

Data Cleaning & Validation (SQL)
Funnel Analysis (Browse → Cart → Checkout → Purchase)
Conversion & Drop-off Analysis
Revenue Analysis (Channel, Device, Region)
User Segmentation (Fast / Medium / Slow converters)

📊 Key Metrics

Metric	Value
Total Users	10,000
Total Revenue	277K
Total Transactions	1,004
Conversion Rate	10.04%
Avg Time to Purchase	10.4 mins

📉 Funnel Insights

Major drop-off at Checkout → Purchase stage
Moderate engagement at Browse → Cart stage
Conversion rate below benchmark (12–15%)

💰 Revenue Insights

Revenue concentrated in key channels
High-value users drive the majority of revenue
👥 Segmentation Insights
⚡ Fast users → High revenue
🐢 Slow users → High drop-off
Enables targeted marketing strategies

🚨 Key Problems

Checkout friction
Low conversion rate
High user hesitation (time to purchase)

🎯 Recommendations

Optimize checkout UX
Improve trust signals (pricing, payment UX)
Retarget abandoned users
Focus on high ROI channels
